Project Folder Structure Guide

A quick reference for the team — where each file goes and why.

Folder Overview

your-project/
  ├── 🚫 .gitignore ← tells Git what not to track (don't touch)
  ├── 🚫 eslint.config.js ← catches code mistakes automatically (don't touch)
  ├── 🚫 index.html ← the one HTML page (don't touch)
  ├── 🚫 package-lock.json ← auto-generated, never edit by hand (don't touch)
  ├── 🚫 vite.config.js ← Vite config (don't touch)
  ├── ✏️ package.json ← only touch when adding new packages
  ├── ✏️ README.md ← update with your project name & description
  ├── 📁 public/ ← favicon and static files (your team works here)
  ├── 📁 docs/ ← team reference docs (this file!)
  └── 📁 src/ ← all your code goes here (your team works here)
      ├── 🚫 main.jsx ← entry point (don't touch)
      ├── 🚫 index.css ← base styles (don't touch)
      ├── ✏️ App.jsx ← connects your pages together
      ├── pages/ ← one file per page
      ├── components/← reusable UI pieces
      ├── assets/ ← images & SVGs
      └── styles/ ← shared CSS files

What Each Folder Is For

📄 src/pages/

One file per page of your website. Each teammate can own a different page file to avoid conflicts.

HomePage.jsx   AboutPage.jsx   ContactPage.jsx   NotFoundPage.jsx

🧩 src/components/

Small, reusable UI pieces used on multiple pages — like a navigation bar or footer that appears everywhere. Name files with a capital letter.

Navbar.jsx   Footer.jsx   Button.jsx   Card.jsx

🖼️ src/assets/

Images, SVGs, and other media that you import directly into your components. Already exists in the Vite template.

logo.png   hero-background.jpg   team-photo.png   icon.svg

📁 public/

Static files that are served exactly as-is — not processed by Vite. The favicon and icon files are already here. Anything you put here is accessible directly by URL (e.g. /favicon.svg).

favicon.svg   icons.svg   robots.txt   og-image.png

🎨 src/styles/

Shared CSS files for the whole project — like a file that defines your team's agreed-upon colors and fonts so everything looks consistent.

variables.css   utilities.css

Quick Team Rules

  • Pull before you start coding every session — git pull first.
  • Each person works on a different page file to avoid merge conflicts.
  • Don't edit main.jsx or vite.config.js without checking with the team.
  • Use descriptive file names — hero-background.jpg, not img1.jpg.
  • Commit small and often — easier to review and easier to undo.
